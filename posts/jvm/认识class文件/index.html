	public String getCompany(){

1.2 JVM虚拟机规范

1.2.1 一个class文件的结构体

虚拟机规范的一个class文件必须按照以下的结构体紧密排列，其中包含了两种类型的数据：无符号数和表。

无符号数：以u表示，后面的数字表示几个字节，可以用来表示数字或者utf8编码构成字符串

表：一般以info结尾，用来表示具有复合关系结构的数据

ClassFile {
    u4             magic; //Class 文件的标志
    u2             minor_version;//JDK 的小版本号
    u2             major_version;//JDK 的大版本号
    u2             constant_pool_count;//常量池的数量
    cp_info        constant_pool[constant_pool_count-1];//常量池
    u2             access_flags;//Class 的访问标记
    u2             this_class;//当前类
    u2             super_class;//父类
    u2             interfaces_count;//接口
    u2             interfaces[interfaces_count];//一个类可以实现多个接口
    u2             fields_count;//Class 文件的字段属性
    field_info     fields[fields_count];//一个类可以有多个字段
    u2             methods_count;//Class 文件的方法数量
    method_info    methods[methods_count];//一个类可以有个多个方法
    u2             attributes_count;//此类的属性表中的属性数
    attribute_info attributes[attributes_count];//属性表集合
}

2. 魔数

cafebabe 用于JVM识别class文件

3. 版本号

通过查表可以确定主次版本号，例如JDK8主版本号在class文件中表示为52 从JDK9开始不再支持编译版本号小于1.5的源码

4. 常量池

常量池格式

1
2
3
4

cp_info { 
    u1 tag; 
    u1 info[]; 
}

主要存放字面量和符号引用

索引从1开始，设计0有特殊作用，可用于表达不引用任何一个常量池

5. 访问标志

  • 这个class是类还是接口
  • 是否为public
  • 是否为abstract
  • 是否被final修饰

6. 当前类、父类、接口

类文件中，这部分用来描述类的继承关系。包括

  • 类索引，指向常量池中一项class_info类型
  • 父类索引，指向常量池中的一项class_info类型或者索引0（Java中有且只有Object类没有父类索引）
  • 接口计数器
  • 接口表，如果有接口的话

单继承特性，所有类都有父类，都指向常量池的某一项，除了Object类，指向0。

7. 字段表

field_info {
    u2             access_flags;
    u2             name_index;
    u2             descriptor_index;

关于字段表需要补充的是，字段表不会列出父类或者父接口继承而来的字段。但是可能出现Java类代码中不存在的字段，例如内部类为了保持对外部类的访问，编译器会自动添加一个指向外部类实例的字段。

7.1 ConstantValue

是指被final static修饰的类型，直接在编译期指向常量，加快运行期效率。

8. 方法表

1
2
3
4
5
6
7

method_info {
    u2             access_flags;
    u2             name_index;
    u2             descriptor_index;
    u2             attributes_count;
    attribute_info attributes[attributes_count];
}

与Class文件中其他的数据要求严格的顺序、长度和内容不同，属性表集合的限制稍微宽松一些，不再要求各个属性表具有严格的顺序。

任何人实现的编译期都可以向属性表写入自定义的属性信息，Java虚拟机运行时会忽略掉它不认识的属性。

8.1 变量槽

  • 可以复用，JVM自动判断变量的作用域来实现变量槽的复用
  • 单个槽32位长度，long和double需要两个变量槽

8.2 Code属性表

Code属性是Class属性中最重要的属性，Code属性用于描述代码，包括字节码。

所有实例方法的参数值从1开始计算，第0位为this，是指向当前对象实例的局部变量

8.3 为什么说栈大小在编译期就已经确定

栈深度和变量槽大小在编译期就已经确定

一个slot长度为32位，保存int、short、reference、byte、char、float和returnAddr，long和double需要两个slot

每个栈帧内部包含一个后进先出的操作数栈，提供给方法计算过程使用
